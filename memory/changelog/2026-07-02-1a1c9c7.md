# v0.1.3：更新入口改良 + BTN 连接状态可视化

## 做了什么

- **header 更新徽标改为跳设置页、自选自动/手动**：以前点「有新版本」直接新开 GitHub Release 页，想自动更新还得自己去设置里翻。现在徽标跳转到设置页「关于 / 检查更新」卡（平滑滚动 + 高亮闪一下），并在该卡同时给出**「⚡ 自动更新（下载+替换+重启）」**与**「⬇ 手动下载（前往 Release）」**两个按钮，由用户自选。轮询与手动检查共用 `renderUpdate`，无论哪条路径到达该卡都能看到版本与两个入口。
- **BTN 连接状态指示器**：设置页 BTN 卡新增一行状态灯——🟢 已连接（显示 abilities 数、黑/白名单/规则组条数、心跳外网 IP、config 更新时间）、🔴 连接失败（附具体错误 + “600s 后自动重试”）、🟡 启动/连接中、⚪ 未启用。保存 BTN 配置后即时刷新，设置页停留时每 3s 轮询。

## 为什么

用户反馈：① 检测到更新后点 header 直接跳 Release、想自动更新必须再进设置，路径绕；希望点 header 直达设置对应位置并能自选自动/手动。② BTN 此前只能看日志判断是否连上，缺一个直观的“能否与 BTN 建立连接”的状态展示。

## 影响的文件 / 范围

- `pbh-btn/src/online.rs`：`BtnState` 新增 `BtnStatus`（config 成败/时间、ability 数、denylist/allowlist/规则组条数、心跳 IP/时间、最近错误+时刻）。
- `pbh-btn/src/lib.rs`：导出 `BtnStatus`；调度循环在 config 拉取成败、规则/黑/白名单更新、心跳、各 ability 失败处写状态；新增 `record_error`。
- `pbh-web/src/routes.rs`：新增 `GET /api/btn/status`（未启用→`enabled:false`；运行中→回全部状态字段）。
- `pbh-web/assets/index.html`：徽标改跳 `#settings`+`gotoUpdate`；关于卡加 `id=aboutCard`、手动下载入口、`renderUpdate`；BTN 卡加状态灯 + `refreshBtnStatus`+`fmtAgo`；`.flash`/`.btnstat`/`a.btn` 样式；接入路由/轮询/保存后刷新。
- 版本 0.1.2 → **0.1.3**。
