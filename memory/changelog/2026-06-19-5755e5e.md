# 删脚本引擎/AutoSTUN，精简结构，文档归位

## 做了什么
1. **完全移除脚本引擎**（Aviator / ExpressionRule，JVM 限定）：删 `pbh-rules` 的 `ScriptEngine` trait 边界提及;
   ExpressionRule 不再列为模块。
2. **完全移除 AutoSTUN / NAT 穿透 / UPnP**：删 `pbh-engine` 的 `NatAddressProvider` + `IdentityNatProvider`;
   peer 直接用下载器返回的原始 `ip:port`，不做 NAT 改写;下载器 trait 精简（去 speed-limiter/BTProtocolPort/natTranslate/traffic-stats 等与封禁无关项）。
3. **架构去冗余**：在 guidelines/02 立「不照搬上游」硬原则（不抄 Spring DI / service-locator / organ 隐喻 / 每 wave 新建池 / 多余抽象）。
4. **文档归位（守则 §1）**：
   - 绑定性「范围/决策」→ `guidelines/01-scope-and-decisions`（权威单一来源）;「架构约定」→ `guidelines/02-architecture`（权威）。
   - `docs/02-strategy-and-roadmap.md` → 改名 `docs/02-roadmap.md`，聚焦 API/里程碑/验收/对拍，范围与架构指向 guidelines。
   - 清理 `docs/01` 内 AutoSTUN/脚本「留接口/stub/JVM sidecar」残留;风险 R5（Aviator）标「已消除」;支撑服务表 Alert/UPnP/AutoSTUN/脚本均标「完全移除」。
   - 全仓交叉引用同步（README、crate 文档注释、启动横幅、test-status），`grep` 确认无死链。

## 为什么
- 用户指示：①直接砍脚本引擎 ②进一步移除 AutoSTUN 这类与封禁 peer 无关的内容 ③优化架构去冗余、不照搬上游
