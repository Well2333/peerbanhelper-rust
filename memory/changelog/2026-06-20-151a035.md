# M6 IPBlackRuleList：IP 黑名单订阅

从 URL 下载社区封禁名单,解析成 CIDR 前缀 trie,命中 peer IP 即封。

## 做了什么
- **解析器 `parse_rule_list`**（纯函数,`pbh-engine/src/ip_rule_list.rs`）：
  - **eMule DAT**：`起始IP , 结束IP , 等级 , 名称`；零填充 IPv4 手工解析（std 拒绝前导零）；等级 ≥128 丢弃；范围用 `ipnet::Ipv4Subnets` 转最优 CIDR 块。
  - **CIDR / 纯 IP**：直接收集（IpMatcher 校验）。
  - 行内注释 `#` 或 `//`（取最先出现者）；空行/纯注释行跳过。
- **`IpBlackRuleList` 模块**：后台 tokio 刷新任务（启动立即一次 + 每 `check-interval`）；订阅来自 `profile.yml` 的 `module.ip-address-blocker-rules.rules`（id→{enabled,name,url}）；下载失败时**保留上一次的内存匹配器**；`Drop` 置停止标志停任务（热重载不泄漏）。`should_ban` 遍历各订阅 trie 最长前缀匹配。
- **`pbh-storage/rule_sub.rs`**：`rule_sub_info`（订阅元信息 upsert/list/delete）+ `rule_sub_log`（更新日志 insert/query）。
- **Web**：`GET /api/sub/rules`（订阅状态：last_update/ent_count）、`GET /api/sub/logs?id=`（更新日志）。
- **接线**：`build_modules` 加 `parse_subs` 解析 rules 子配置；`pbh-engine` 加 `reqwest`/`ipnet` 依赖。模块**默认关闭**（opt-in）。

## 为什么
- 社区维护的 IP 黑名单（如 all-in-one）是高价值的反吸血/反恶意手段。订阅式自动拉取让封禁库持续更新。
- 订阅配置放 profile.yml（与上游一致）：保存 profile 即重建模块触发即时下载,无需额外的句柄/通知线程化。

  - 测后恢复原 profile；60s 封禁自动过期,**qB banned_IPs 清零**。
