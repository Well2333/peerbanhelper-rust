# M2 下载器 qBittorrent + qBittorrentEE

## 做了什么
- `Downloader` trait（`async_trait`，v2 精简）：`login / get_torrents / get_peers / apply_ban_list / feature_flags / is_paused`。
- `build_downloader` 工厂（按 `type` 分发，仅 qb/qbee，保留可扩展）。
- `QBittorrentClient`（`reqwest`，cookie_store + gzip + rustls）：
  - 登录：密码模式（`/auth/login` → "Ok."）与 api-key Bearer 模式;`/app/version` 解析刷新 `RANGE_BAN`(≥5.3.0);
    登录副作用 `enable_multi_connections_from_same_ip=false`;EE shadowban 需服务端启用（`/app/preferences.shadow_ban_enabled`）。
  - 拉取：分页 `/torrents/info?filter=active`（去重、按 `ignore_private` 过滤）;`/sync/torrentPeers` 解析 peer
    （过滤 Web/onion/i2p/空 ip、EE shadowbanned;raw_ip 取下载器 map 键）。
  - 封禁：全量 `/app/setPreferences`(`banned_IPs`/EE `shadow_banned_IPs`) 与增量 `/transfer/{banPeers,shadowbanPeers}`;
    分发逻辑 `!apply_full && increment_ban && !added.empty`。
  - HTTP basic-auth（反代）、`verify-ssl=false` 关闭证书校验。
- 纯逻辑封禁串拼装抽成可测函数：`join_full_ban_string`（规范化/compressed/去重/换行/CIDR 门控/单主机）、`join_increment_peers`（`|` 分隔）。

## 为什么
- M2 是面向真实下载器的关键路径,也是 M3 流水线的数据源与封禁出口;封禁串须与上游字节级一致,故抽成纯函数先测。

## 影响的文件 / 范围
- pbh-downloader：Cargo(reqwest/serde/async-trait/ipnet/semver) + lib.rs(trait/类型/工厂/封禁串) + qbittorrent.rs(客户端)。

## 注意事项 / 后续
- **HTTP 交互未对真实 qB 验证**（环境无 qB 实例）——见待测报告 M2;封禁串与上游逐字节对拍亦待真机/录制。
- `completed_size` 暂为 -1（M5 PCB 经 `/torrents/properties` 补全）。
- UA 用 `PeerBanHelper-Rust/<ver>`（非上游字节一致,仅 qB 日志可见,不影响封禁）。
- `DownloaderManager`（持久化配置/列表/增删）放到 M3 装配时实现。
- 45 单测全绿;clippy+fmt 零告警。下一步：M3 流水线 + 调度 + BanManager。
