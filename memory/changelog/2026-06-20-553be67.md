# M8 BTN（BitTorrent Threat Network）云端威胁情报

跨 PBH 实例共享封禁/规则的云端网络。`crates/pbh-btn/`。

## 做了什么
- **`pow.rs`**：PoW captcha 求解（找 nonce 使 `SHA256(challenge||nonce)` 前导 N 个 0 比特）+ 难度位检查。单测求解低难度挑战。
- **`hash.rs`**：种子隐私哈希 `getHashedIdentifier`（CRC32-IEEE 小端 hex 作 salt + `SHA256(lower(hash)+salt)`）；从不上报原始 infohash。单测 CRC32 已知向量。
- **`model.rs`**：serde 模型——config 响应（ability map + legacy 判定）、ruleset（peer_id/client_name/ip/port 分类）、SubmitBans 请求体。
- **`client.rs`**：`BtnClient`——固定头注入（X-BTN-AppID/Secret + BTN-* + `Authentication: Bearer id@secret`；匿名用 X-BTN-InstallationID）、config 拉取、`fetch_rules`(`?rev=`/204 未变)、`fetch_ip_list`(X-BTN-ContentVersion)、`submit_gzip`（gzip 上行）。
- **`online.rs`**：`BtnNetworkOnline` 规则模块——短路 AllowList→SKIP / DenyList→BAN / Rules(peer_id→client_name→ip→port)→BAN；共享 `BtnState`（调度器写、模块读）；`apply_ruleset/denylist/allowlist`。
- **`lib.rs`**：`spawn` 启动 `BtnManager`——拉 config（失败 600s 重试）→ 按各 ability interval 下行（rules/denylist/allowlist 更新状态）+ 上行 `submit_bans`（history.id 游标分页 + gzip + KV 续传）；停止标志。
- **接线**：`build_modules` 加 `btn_state` 参数（启用即推 `BtnNetworkOnline`）；`main` 据 `config.yml` `btn.enabled` 启动调度并存 `btn_state`；`WebState` 加 `btn_state`（热重载重建）；`storage::query_btn_bans`（join torrents 取 info_hash/size）。模块**默认关闭**（需 app-id/secret）。

## 为什么
- 用户明确要求「保留完整 BTN（下行+上行）」。BTN 是跨实例威胁情报共享,大幅增强封禁覆盖。

- `cargo clippy -D warnings` 零告警;`cargo fmt --check` 干净;`cargo test --workspace` **85 单测全绿**（新增 BTN 13：pow 2 / hash 2 / model 4 / online 2 / client 3）。
- **运行时冒烟**（release 实跑）：config.yml 启用 BTN + 假 URL → 日志 `BTN 已启用,后台调度启动`、modules=5（含 BtnNetworkOnline）、config 拉取失败优雅 `600s 后重试`、**进程存活无崩溃**。测后恢复 config.yml。

## 注意事项 / 后续（重要）
- **未对真实 BTN 服务端联网验证**：需用户的 `app-id`/`app-secret` + BTN 账号。本实现以单测覆盖 PoW/哈希/序列化/规则应用逻辑;协议字节级一致性（时间格式、structured_data 字符串化、InetAddress 串形）按上游分析实现,**真实对拍待用户提供 BTN 凭证**（风险登记册 R2）。
- 简化项:① BTN peer_id/client_name 模式按 CONTAINS 近似（上游 RuleParser 语义）;② IP 名单解析仅 CIDR/纯 IP（不处理 DAT 范围）;③ 仅实现 submit_bans 上行（submit_swarm/history 留待，需 peer_records/tracked_swarm 采集）;④ PoW captcha 头注入实现但调度未自动获取挑战（多数 ability `pow_captcha:false`）;⑤ reconfigure 用 config 1h 老化重拉近似;⑥ legacy(<20) 分支未单独实现（现代协议为主）。
- 种子哈希的 Guava CRC32 字节序按小端 hex 实现,真实对拍待 BTN 服务端确认。
- 至此 **M8 核心闭环就绪**（下行消费威胁情报→封禁 + 上行 submit_bans），M0-M8 全部里程碑代码完成（M7=WebUI 已于阶段 A 重做）。
