# v0.1.4：代理审计与重构 —— 启动即生效 + 分类走代理

## 做了什么

审计并统一了**全部出站请求**的代理走向，修复"启动时代理没生效、检查更新等没走代理"。

- **代理权威化（修启动优先级）**：`pbh-net::build_client` 改为**「配置了代理就始终应用」**，去掉原先"探测不可达 → 回退直连"的逻辑。此前 pbh 若先于 clash/v2ray 启动，1s TCP 探测失败会**永久直连**、直到重存配置才恢复——这正是日志里 BTN 一批 `error sending request` 后直到 `00:49 出站代理已启用` 才恢复的成因。现改为始终走代理，各处重试循环（BTN 30s/600s、GeoIP 退避、订阅）在代理就绪后自动恢复；仅对"已配置但当前探测不可达"warn 一次提示。
- **分类代理开关**：`config.yml` 的 `network` 新增 `proxy-targets`（`btn` / `geoip` / `rule-subscription` / `update` / `public-ip`），**默认只勾选易被墙/审查的境外目标**（btn、geoip、订阅、更新），本机公网 IP 探测（用境内服务）默认直连。新增 `NetworkConfig::proxy_for(ProxyTarget)` 供各处统一取用。
- **所有联网点接入分类门控**：BTN（`btn_manager`）、IP 规则订阅（`build_modules` ×3 处）、GeoIP（启动后台循环 + 手动更新）、检查更新 / 一键自更新、本机公网 IP（`netinfo`，此前写死直连）。**下载器连接仍恒不走代理**（本地）。
- **设置页「网络代理」卡**新增 5 个分类勾选框；`GET/PUT /api/config/app` 读写 `proxy-targets`，保存后热生效（BTN 重启 + 模块重建），并即时刷新 BTN 状态灯。

## 为什么

用户反馈：① BTN config 能拉到但规则/心跳失败、检查更新也没走代理，怀疑代理应用不统一、启动时机太晚；要求提高代理启动优先级、确保所有涉网处都能走代理。② 希望设置页能自由控制哪些走代理，默认只勾可能被拦截/审查的请求。

## 影响的文件 / 范围

- `pbh-config`：`NetworkConfig` 加 `proxy_targets: ProxyTargets` + `proxy_for(ProxyTarget)`；新增 `ProxyTargets`（默认境外目标 on、public-ip off）与 `ProxyTarget` 枚举；lib 导出。
- `pbh-net`：`build_client` 代理权威化（始终应用配置的代理，不再回退直连）。
- `pbh-server/main.rs`：启动 build_modules → `RuleSubscription`；GeoIP 循环 → `Geoip`。
- `pbh-web/btn_manager.rs`：BTN → `Btn`。
- `pbh-web/routes.rs`：3 处 build_modules → `RuleSubscription`；geoip_update → `Geoip`；update_check/apply_update → `Update`；netinfo → `PublicIp`；`get/put_app_config` 读写 `proxy_targets`。
- `pbh-web/assets/index.html`：代理卡 5 个分类勾选框 + 加载/保存 + 说明更新。
- 版本 0.1.3 → **0.1.4**。

## 注意事项 / 后续

- 冒烟验证：`GET /api/config/app` 返回默认 targets（btn/geoip/订阅/更新=true、public-ip=false）；PUT 持久化并以 kebab-case `proxy-targets` 落盘、回读一致。
- 新增 `proxy_for` 门控/默认值/kebab 往返 3 个用例；全工作区 `cargo test --workspace` 21 套件全绿、`clippy --all-targets -D warnings` 零告警。
- **行为变化提示**：配置了代理但代理实际不可用时，不再自动回退直连（各类别按勾选走代理）。若想某类别直连，取消其勾选即可。
