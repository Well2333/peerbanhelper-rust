# 战略转向 v2（极简重构）—— 弃用 Vue 前端、自研极简 API

## 做了什么
- 确立 v2 方案并落成权威文档 `docs/05-revised-strategy.md`:从「忠实移植 + 复用 Vue 前端」
  转为「完全重构、最精简」,目标仍是与原版一致的封禁效果。
- 据用户 4 项新决策定稿范围:
  1) 效果=封禁判定基本一致 → 保留全部高价值规则;
  2) BTN 完整保留(下行+上行);
  3) 弃用现有 Vue 前端 → 自研极简 REST API + 内置轻量单页(rust-embed);
  4) 保留封禁历史+日志。
- 砍除:整套 Vue 前端及其 API 契约、i18n、图表/分析、推送(8 通道)、Alert(降级为日志)、
  表 traffic_journal_v3 / peer_connection_metrics(+track) / alert。
- 文档:docs/02、03 加「已被 docs/05 取代」横幅并改标题为参考;docs/04 标注精简表集并给 4 表打 🔻。
- 骨架同步:移除 `pbh-i18n`、`pbh-notify` 两 crate(10 crate);`pbh-web` 信封改 `{ok,data,error}`、
  角色简化为 Anyone/Authed;更新根 Cargo.toml(去 lettre/pulldown-cmark,加 rust-embed)、README、启动横幅。

## 为什么
- 用户改变主意:要最精简的重构、与原版一致的效果。其选择实际把"精简"集中在**弃用沉重前端**
  (免去复刻 ~90 端点/Gson/SPA/OOBE/fail2ban 与整条 pnpm/Vite 工具链),而封禁能力与 BTN 仍要全。
