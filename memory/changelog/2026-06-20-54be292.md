# WebUI 多页改版 + 运行统计 + profile 热重载端点

## 做了什么

### 前端（`crates/pbh-web/assets/index.html` 全量重写）
参考原版 PBH（ArcoDesign 风格）重做内置 vanilla 单页:
- **顶栏导航**（仪表盘/封禁列表/封禁历史/下载器/规则配置/实时日志）+ hash 路由 + 状态指示。
- **深浅主题**切换（CSS 变量,localStorage 记忆）。
- **仪表盘**:统计卡片网格（检查 peer / 封禁 / 解封 / 当前封禁 IP / Ban Wave 轮数 / 模块数 / 下载器 / 间隔）+ **下载器卡片**（在线/离线/未知圆点、类型徽章、暂停标记、编辑/删除）。
- **封禁列表**:搜索过滤 + 卡片式（IP:port、模块、规则、原因、封禁/解封时间、地址、地理占位）+ 手动封禁（可填时长）+ 单个/全部解封。
- **封禁历史**:表格 + 分页(上/下页)。
- **下载器**:表单（增删改 + 测试连接）+ 列表。
- **规则配置**:`profile.yml` 文本编辑器 + 保存即生效 + 提示。
- **实时日志**:着色级别、自动滚动、500 行上限。
- toast 提示、空状态、响应式栅格。

### 后端
- `pbh-engine::BanManager`:新增运行统计 `Stats`（原子计数:checked_peers/banned_peers/unbanned_peers/waves/last_wave_at/last_wave_ms）+ `stats()` 快照;每下载器登录状态 `login_status`（id→bool）+ `downloader_status()`;`module_count()`。
- **模块热重载**:`modules` 由 `Vec` 改 `RwLock<Vec<…>>`;`run_modules` 读锁迭代;新增 `rebuild_modules()`。
- `pbh-config`:新增 `save_profile()` 函数 + `ConfigHandle::save_profile()/save_app()`（写盘 + 热重载）。
- `pbh-web`:`/api/status` 扩展（stats + downloader_list 含在线状态 + modules）;新增 `GET/PUT /api/config/profile`（YAML 文本收发,PUT 校验→保存→热重载→`build_modules` 重建,**即时生效无需重启**)。新增依赖 `serde_yaml`。

## 为什么
用户要求「参考 qbh 原始设计,把网站做得好看、功能更全」。原 UI 为单页堆叠,信息密度低、无主题、无规则配置入口。本次对齐原版的多视图结构与视觉语言,并补齐仪表盘统计与在线规则编辑能力。

## 影响的文件 / 范围
- 改:`pbh-web`(index.html 重写、routes status/profile、Cargo serde_yaml)、`pbh-engine`(ban_manager Stats/热重载、lib 导出 StatsSnapshot)、`pbh-config`(save_profile)。
- 无破坏性变更;旧 API 端点保持兼容。
