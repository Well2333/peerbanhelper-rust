# M0 地基 —— 配置 / SQLite / 日志 / AppContext + 工具链

## 做了什么
- **pbh-config**：`config.yml`/`profile.yml` 的 serde 模型（kebab-case，自定义默认）、加载(缺失写默认)、
  `tokio::sync::watch` 热重载（`ConfigHandle`）、目录解析（`Paths`：data/config/persist/logs）、`save_app`。
- **pbh-storage**：`sqlx` SQLite —— WAL/synchronous=NORMAL/busy_timeout=60000/mmap_size、写池 `max_connections(1)`;
  `migrations/0001_initial.sql`（v2 精简 10 表）;`metadata` KV(get/set)。
- **pbh-domain**：新增 `log` 模块（`LogBuffer` 环形缓冲 + `LogEntry`，供后续 WS 日志流，std-only）;
  `PeerAction` 改 `#[derive(Default)]`。
- **pbh-server**：异步 `main` 装配 —— 目录→tracing(控制台+按日文件+缓冲层)→配置(首启生成 token 写回 config.yml)→
  SQLite 建库+迁移→安装ID(KV)→打印状态→干净退出;`AppContext` 组合根;`logging` 模块。
- **工具链**：环境系统 cargo 1.75 无法编译现代依赖(hashbrown 0.17 等需 edition2024/≥1.85) →
  安装 rustup `stable`(1.96)，构建用 `~/.cargo/bin/cargo`;workspace `rust-version`→1.85。
- **附带**：本提交还含用户把《最高优先级工作守则》由 `docs/` 移到 `memory/`(docs/ 现空)。
  其遗留引用在后续提交修正。

## 为什么
- 用户要求开始 M0 并持续推进。M0 是后续一切的地基（配置/存储/日志/装配）。
- 工具链升级是硬前置：现代 Rust 依赖普遍要求 ≥1.85，系统 1.75 无法编译。

## 影响的文件 / 范围
- 新增：pbh-config/{model,paths}.rs、pbh-domain/log.rs、pbh-server/{context,logging}.rs、storage migrations。
- 改：各对应 Cargo.toml、根 Cargo.toml(rust-version)、Cargo.lock、main.rs、ban.rs。
- 文档：roadmap M0 标完成;guidelines/03 工具链与依赖政策;test-status。
