# fix: WebUI 缺失 `.hidden` CSS 导致登录框与主界面同时显示

## 问题
用户截图发现:已登录时,登录框仍显示在仪表盘下方（两者同时可见）。

## 根因
阶段 A 重写 `index.html` 时,**漏掉了 `.hidden { display:none }` CSS 规则**。
而 `#topbar`/`#app`/`#login` 三者都用 `class="hidden"` + JS `classList.toggle('hidden', ...)` 控制可见性。
`.hidden` 无定义 → 加上该 class 不起任何作用 → 登录框（及初始隐藏的 app）始终可见。

## 修复
补回 `.hidden { display:none !important; }`。

## 验证（CDP / Chrome headless 实测各页面）
用 `websockets` + google-chrome headless 驱动 CDP,逐页截图 + 计算样式断言:
- **未登录**：`login.display=block`、`app.display=none`（登录框单独居中,主界面隐藏）✓
- **登录后**：`login.display=none`、`app.display=block`、`topbar.display=flex`（登录框正确隐藏）✓
- **各页面**（仪表盘/封禁列表/封禁历史/下载器/规则配置/实时日志）：`page-on=true`、`login.display=none`,渲染正常,无遮罩 ✓
- 截图目视确认:仪表盘统计卡片/下载器卡片、规则页 profile.yml 编辑器+订阅状态表、封禁列表搜索/手动封禁、登录页单独居中——均正常。

## 注意
- 复测方法（无 node/playwright,有 google-chrome）：`websockets` 库 + CDP（`Page.captureScreenshot`/`Runtime.evaluate`），脚本在 job tmp。后续前端回归可复用。
- 教训:重写含 class 切换的页面时,务必确认对应 CSS 工具类（`.hidden` 等）一并保留。
