# ci: GitHub Actions —— CI + 发版多平台自动构建

## 做了什么
- **`.github/workflows/ci.yml`**：push `main` / PR 触发 → `cargo fmt --check` + `cargo clippy -D warnings` + `cargo test --workspace`（dtolnay/rust-toolchain stable + Swatinem/rust-cache）。
- **`.github/workflows/release.yml`**：push `v*` 标签触发（或手动 dispatch）→ 矩阵构建 4 平台:
  - `x86_64-unknown-linux-gnu`（ubuntu）→ tar.gz
  - `x86_64-pc-windows-msvc`（windows）→ zip
  - `aarch64-apple-darwin`（macos-latest）→ tar.gz
  - `x86_64-apple-darwin`（macos-13）→ tar.gz
  - 每平台 `cargo build --release -p pbh-server --target <t>`,打包(含 README),上传 build artifact;**标签触发时附加到对应 Release**（softprops/action-gh-release）。

## 为什么
- 用户要求「发版时自动构建可执行程序」。CI 保证每次 push 质量门(clippy/fmt/test);release 工作流让打 tag 即产出各平台二进制并挂到 Release。

## 注意
- 本项目用 `sqlx` 运行时查询(`query`/`query_as`)+ `migrate!`(路径内嵌),**非 `query!` 宏**,故 CI/构建**无需 DATABASE_URL / 离线缓存**。
- rust-version=1.85;runner stable 远高于此,满足 edition2024 依赖。
- 未做 GUI/code-sign/notarize(macos 二进制未签名,用户首次运行需放行)。
- 工作流仅本地校验了 YAML 语法,真实运行待用户 push 到 GitHub 验证。
