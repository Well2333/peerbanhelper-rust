# M5 ProgressCheatBlocker：持久化（载入/批刷/8h 清理/解封重置）

PCB 状态落库,重启续算。M5 至此完成。

## 做了什么
- **`pbh-storage/src/pcb.rs`**:`PcbAnalysis`/`PcbAddressRow`/`PcbRangeRow` + 批量 upsert（单事务,`ON CONFLICT` 更新分析列、保留 first_time_seen）+ 按 `last_time_seen >= since` 载入 + `cleanup_pcb(cutoff)`。`downloader` 字段统一空串（单下载器简化）。
- **`ProgressCheatBlocker::with_persistence(cfg, db)`**:后台 tokio 任务——首次从 DB 载入近期状态 → 每 60s 批刷脏条目 → 每 8h 清理过期。任务只持有缓存/db/停止标志的克隆（非模块 Arc）；`Drop` 置停止标志并终刷，**热重载替换模块不泄漏任务**。
- **解封重置**:`RuleFeatureModule` 加 `on_unban(ip)` 钩子（默认空实现）；PCB 实现为清除该 IP 的单点跟踪缓存（给重新开始机会,段聚合保留）；`BanManager` 在到期解封 + 手动解封时通知各模块。
- **接线**:`build_modules` 加 `db: &Db` 参数（main / web put_profile 同步传入）；`PcbConfig` 加 `enable-persist`(true)/`persist-duration`(14d)；启用持久化时走 `with_persistence`，否则纯内存。

## 为什么
- M5 验收含「持久化重启续算」。PCB 跟踪的上传累计若每次重启清零,吸血者重连即可绕过——持久化是反吸血有效性的关键。

## 注意事项 / 后续
