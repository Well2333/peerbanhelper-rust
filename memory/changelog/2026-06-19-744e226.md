# 构建脚本 build.sh + 加速 release 配置

## 做了什么
- 新增 `build.sh`（可执行），子命令：
  - `build`（默认，release）/ `debug` / `run` / `test` / `clippy`(`-D warnings` + fmt) / `package` / `clean`。
  - 自动优先用 `~/.cargo/bin/cargo`，校验版本 ≥ 1.85，缺失则给 rustup 安装提示。
  - `package`：release 构建后打包 `dist/pbh-<ver>-<os>-<arch>.tar.gz`（含二进制 + README + `downloaders.yml.example` + `运行说明.txt`）。
- 放宽 `profile.release`：去掉 `lto="thin"` 与 `codegen-units=1`，仅保留 `opt-level=3` + `strip`。release 构建从「数分钟」降到 **~80s**，产物 11M 单文件（内嵌 Web UI）。
- `.gitignore` 加 `/dist`；README 增「构建与测试」(build.sh 用法)。

## 为什么
- 用户需要「编译脚本 + 编译成品」来构建测试。脚本封装了工具链选择（避开系统 1.75）、构建、测试、打包;加速 release 让出成品更快。

## 影响的文件 / 范围
- 新增 build.sh;改 Cargo.toml(profile.release)、.gitignore、README、test-status。
- `dist/` 产物不入库。

## 注意事项 / 后续
- **构建务必用 rustup 的 cargo**（脚本已自动处理）;系统 1.75 会因 edition2024 依赖失败。
- 已产出并实跑验证：`target/release/pbh`（11M）+ `dist/pbh-0.1.0-linux-x86_64.tar.gz`（4.6M）;`GET /`/登录/status 均正确。
- 需要极致体积/性能时，临时在 profile.release 加回 `lto="thin"`、`codegen-units=1`。
