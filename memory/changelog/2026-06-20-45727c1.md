# feat: banlist 持久化(快照/恢复) + SIGTERM 优雅关闭

## 修复的真缺口
`banlist` 表早已存在但**从未读写** → 进程重启**丢失内存封禁**(规则命中/手动封禁;PCB 状态此前已持久化但封禁本身没有)。订阅/BTN 会重新封回,但手动封禁与短时规则封禁会丢。

## 做了什么
- **`pbh-domain`**：`BanMetadata` / `PeerAddress` 加 `serde::Serialize/Deserialize`(引入 serde 依赖)。IpAddr 用 serde 默认串形。
- **`pbh-storage`**：`save_banlist`(事务全量替换)+ `load_banlist`(读 (网络字符串, BanMetadata JSON))。
- **`BanManager`**：`snapshot_to_db`(序列化 `ban_list.snapshot()` 落库)、run_once 末尾**每小时**自动快照、`restore_banlist`(关联函数,启动把未过期封禁恢复进内存 BanList)。
- **`main`**：启动后 `restore_banlist`；关闭前 `snapshot_to_db`;`shutdown_signal` 同时处理 **Ctrl-C + SIGTERM**(systemd/docker stop 发 SIGTERM),`#[cfg(unix)]`/`#[cfg(not(unix))]` 分别实现。

## 验证
- clippy/fmt 干净;`cargo test --workspace` **94 单测全绿**(banlist save/load 全量替换断言并入既有 storage 测试)。
- **真机端到端**：手动封 `198.51.100.200`(24h)→ SIGTERM(日志「正在关闭…」,`banlist` 表落库 6 行含该 IP）→ 重启日志 **「恢复 6 条封禁快照」**,该 IP 仍在封禁列表。**快照→恢复闭环跑通**。

## 注意
- 快照为全量替换(每小时 + 关闭)。崩溃(SIGKILL/断电)且距上次快照 <1h 的封禁会丢(可接受;订阅/BTN/规则会重新封)。
- SIGKILL(`kill -9`)不触发快照(无法捕获),属预期。
