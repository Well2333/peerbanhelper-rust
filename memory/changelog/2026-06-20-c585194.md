# M5 ProgressCheatBlocker：进度作弊检测核心判定状态机（内存）

反吸血核心模块的判定逻辑。`crates/pbh-engine/src/pcb.rs`。

## 做了什么
- **判定状态机 `evaluate()`**（纯函数,便于序列回放测试）：
  - 上传增量计算（处理重连/重置/-1：报告值变小则取报告值本身,clamp ≥0）。
  - `computedUploaded = max(报告值, 单IP累计, 段累计)`——重连归零也无法逃避。
  - 四道**严格短路**子检查:
    1. **fast-pcb-test**:`computedUploaded >= pct*size` → `BAN_FOR_DISCONNECT`（短封强制断连复测,一次性,默认 10%/15s）。
    2. **excessive**:Case1 上传 > 种子大小×倍率;Case2 上传 > 已完成量×倍率（默认 1.5）→ `BAN`。
    3. **difference**:推算进度 − 自报进度 > `maximum-difference`(0.1) → **ban-delay 窗口**（默认 30s）;首超标排程窗口、窗口内放行、到期仍超标才 `BAN`。
    4. **rewind**:`max(last_report) − 自报进度 > rewind-max`(0.07) → `BAN`（progress>0 或窗口到期）。
  - finalize（始终执行,对应上游 `finally`）:刷新 last_report_uploaded/progress、last_torrent_completed_size(取 max)、last_time_seen、dirty。
- **双视图状态**:单 IP（`pcb_address`）+ 前缀段（`pcb_range`,IPv4 /32、IPv6 /56,跨 IP 聚合）防换 IP 绕过。ban-delay 窗口同时作用于两者。
- **模块封装** `ProgressCheatBlocker`:moka 缓存（addr_cache/range_cache, key=`torrent_id|ip:port` 与 `torrent_id|prefix`）+ 统一加锁顺序（addr→range）。
- `PcbConfig`（14 项,默认值对齐上游）+ build_modules 构建（**默认启用**——旗舰反吸血功能）。

## 为什么
- PCB 是 PeerBanHelper 的核心反吸血能力,用户要求保留「原版一致效果」。默认启用。
- 拆成「内存核心 + 后续持久化」两步提交,先交付经过测试的判定逻辑,降低大改风险。
