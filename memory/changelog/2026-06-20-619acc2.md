# fix: 参数路由 axum 0.7 :param（修复 404）+ WS 实时日志流

## 重大 Bug 修复
- 实际依赖 `axum=0.7.9` / `matchit 0.7`（用 `:param` 语法），但 M7 路由全写成 `{param}`（axum 0.8 语法）。
- 后果：`/api/bans/{ip}`、`/api/downloaders/{id}` 被当**字面量路径**,真实 ID 永远 **404** → **手动解封、删除下载器实际不可用**（此前观察到的"解封"其实是封禁自然到期）。
- 修复：改为 `:ip` / `:id`。真机验证 `DELETE /api/bans/203.0.113.99 → HTTP 200`（此前 404）。

## WS 实时日志流（替代 3s 轮询）
- 后端 `GET /api/logs/stream`：`WebSocketUpgrade` + query token 鉴权 + `offset` 回放；700ms 轮询环形缓冲推送新增日志;`tokio::select!` 处理客户端关闭。
- 前端：改用 `WebSocket`（断线 3s 自动重连）替代每 3s 的 `/api/logs` 轮询;日志即时推送。
- 真机验证：WS 连接收到回放日志。

## 注意
- 教训:写路由时确认依赖的 axum/matchit 版本对应的参数语法（0.7=`:p` / 0.8=`{p}`）。若将来升级 axum 到 0.8,需把 `:param` 改回 `{param}`。
- WS 仍是服务端 700ms 轮询环形缓冲(非 broadcast 推送),但省去客户端 HTTP 轮询;真 push 需给 LogBuffer 加 broadcast,留待。
