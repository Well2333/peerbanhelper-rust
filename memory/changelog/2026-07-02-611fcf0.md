# 修复 qB 白名单免鉴权下"测试连接报错 + 下载器被静默挂起"

## 做了什么

- **兼容 qB「跳过身份验证」白名单**：下载器空账密、靠 qB「对本机/白名单子网跳过身份验证」免密访问时，qB 的 `/api/v2/auth/login` 仍按账密返回 `"Fails."`。`login()` 现在收到非 `"Ok."` 时不再直接判死，而是用需鉴权的 `/app/version` **探活**——能取到版本 → 会话有效（绕过生效）→ 按已登录继续；探活也失败才判真失败。
- **测试连接详细报错**：`test_downloader` 用 `err_chain` 展开错误的完整 `source` 链，把 TLS 握手 / 证书校验 / 连接被拒 / 超时 / DNS 等**底层真实原因**透传前端；登录被拒时给出**可操作提示**（区分"账密错"与"未开白名单免鉴权"，指向 qB「选项 → Web UI」设置）。

## 为什么

用户反馈：点「测试连接」返回 `{"success":false,"message":"登录被拒: Fails."}`，但账密（实为空、靠白名单免鉴权）正确且能连上。定位发现两处影响：① Web 测试按钮误报；② **更严重**——`ban_manager` 每轮 `login()` 都判失败，命中"登录被拒即暂停"分支，**把下载器一启动就挂起**，实际根本没在封禁 peer。根因是把 qB 登录接口对空账密的 `"Fails."` 当成了硬失败，而没考虑白名单绕过场景（qB 登录接口独立校验账密，与其它接口的绕过鉴权是两回事）。

## 影响的文件 / 范围

- `pbh-downloader/src/qbittorrent.rs`：`login()` 增加 `/app/version` 探活容错；新增 `login_reject_hint` 组装可操作提示；新增 3 个测试（tokio mock qB：绕过成功 / 无绕过失败带提示 + 文案断言）。
- `pbh-web/src/routes.rs`：新增 `err_chain`（展开错误 source 链），`test_downloader` 用它输出详细原因。
