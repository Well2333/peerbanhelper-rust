# M6 IPBlackList + GeoIP 可选注入

## 做了什么
- **`pbh-geoip`（落地）**：`MaxmindProvider` 读标准 MaxMind City + ASN mmdb（City→country_iso/country_name/city_name，ASN→asn/as_organization）；`load_from_dir` 约定从 `<data>/geoip/`（`GeoLite2-City.mmdb` / `GeoLite2-ASN.mmdb` 等）加载;`IpGeoData` 加 `Serialize`。**可选注入**：无 mmdb → `MaxmindProvider::load` 返回 None → 降级。
- **`IpBlackList` 模块**（`pbh-engine/src/ip_blacklist.rs`）：短路 port → ip/CIDR → asn → region → city → net-type。port/ip 不依赖 GeoIP;asn/region/city/net-type 仅在 GeoIP 可用时检查,否则跳过（pass）。
- **接线**：`build_modules` 加 `geoip: &Option<Arc<dyn GeoIpProvider>>` 参数 + `int_list`/`str_list`/`enabled_net_types` 配置助手;`main` 从 `<data>/geoip` 加载 provider 并传入 + 存进 `WebState`（供 profile 热重载重建）。模块**默认关闭**。

## 为什么
- 完成 M6「GeoIP + IP 黑名单族」。IPBlackList 让用户按 IP/端口/地区/ASN 精确封禁。GeoIP 作可选能力,有 mmdb 即增强,无则不影响 ip/port。

## 验证
- `cargo clippy -D warnings` 零告警;`cargo fmt --check` 干净;`cargo test --workspace` **72 单测全绿**（新增 IPBlackList 2：ip/port 无 GeoIP 封禁、无 provider 时地理检查跳过）。
- 本环境无 mmdb：GeoIP provider = None,asn/region/city/net-type 检查降级跳过,符合设计。ip/port 路径单测覆盖。

## 注意事项 / 后续
- **GeoCN 未移植**（中国网络类型 net-type / 行政区划 cn_province/cn_city）：需 GeoCN 数据库 + 行政区划 CSV,本环境无数据,字段保留恒 None;net-type 配置可读但不命中。
- GeoIP 真机增强未验证（用户无 MaxMind 授权文件）；放 `<data>/geoip/*.mmdb` 即生效。
- 封禁历史的 `peer_geoip` 列回填、WebUI 地理展示留待阶段 F。
- 至此 **M6 完成**：IPBlackRuleList（订阅,已真机验证 628 条/24 命中）+ IPBlackList + GeoIP 可选注入。
