# feat: 订阅增删改(WebUI + 后端端点)

## 做了什么
- **后端**(`pbh-web/routes.rs`):
  - `PUT /api/sub/rules` {rule_id,name,url,enabled}：写入 `profile.yml` 的 `module.ip-address-blocker-rules.rules`(并确保模块 enabled),save_profile + 重建模块 → IpBlackRuleList 立即后台下载。校验 rule_id 非空/不含 `.`、url 非空。
  - `DELETE /api/sub/rules/:id`：从 profile 移除该 rule + `db.delete_rule_sub` + 重建;**删到空则把模块 enabled 设 false**(避免残留空 IPBlackRuleList)。
  - `GET /api/sub/rules` 改为**以 profile 配置为准**(id/name/url/enabled)合并 DB 状态(ent_count/last_update)。
  - serde_yaml Mapping 操作助手 set/remove_sub_rule + save_and_rebuild。
- **前端**:订阅面板加增删改表单(规则ID/名称/URL/启用复选)+ 每行「编辑/删除」按钮;保存即提示后台下载,2.5s 后刷新列表。

## 验证
- clippy/fmt 干净;`cargo test --workspace` **94 单测全绿**。
- **真机端到端**:PUT `test-aio`(all.txt)→ modules=6 + 下载 **626 条** + 列表显示;DELETE → 移除(订阅数 0、profile.yml 无残留、模块回 5)。profile.yml 正确读写。

## 注意
- 订阅仍以 profile.yml 为单一事实源(规则页 YAML 编辑与此 UI 等价);DB 仅存下载状态/日志。
- 新增订阅用模块默认 ban-duration(全局 14 天);如需自定义时长仍走 YAML 编辑。
